"""
【方法1：BBox索引】
原理：        核的边界框预计算 → 只查询空间相交的核
什么是BBox？  一个矩形 (y_min, y_max, x_min, x_max) 包围所有核像素
检查流程：
  1. 细胞BBox: (100, 200, 150, 300)
  2. 核BBox:   (90, 150, 140, 280)  ← 与细胞相交
  3. 核BBox:   (300, 400, 400, 500) ← 不相交，跳过
  4. 核BBox:   (110, 180, 160, 290) ← 与细胞相交
优势：   简单直接，内存占用少
劣势：   要遍历所有核-细胞对，核数过多时仍慢


【方法2：SciPy ndimage】
原理：        用C语言库加速标签查询和面积计算
关键函数：    ndimage.find_objects(), ndimage.sum()
检查流程：
  1. find_objects() - 一次性获取所有标签的BBox （C代码，很快）
  2. 在BBox范围内计算重叠 （利用Python + NumPy向量化）
  3. 找最大值 （NumPy的argmax）
优势：   通用优化，SciPy库已编译，无需修改太多代码
劣势：   不如完全向量化快


【方法3：完全向量化】
原理：        构建m×n矩阵，矩阵[i,j]=核i与细胞j的重叠像素数
            然后直接用矩阵运算找每个细胞的最佳匹配
矩阵示例：
  核\细胞   C1   C2   C3   C4   C5
  N1       0    125  0    0    0     ← N1与C2重叠125像素
  N2       80   0    200  0    0     ← N2与C3重叠200像素
  N3       0    0    0    150  0
  ...
检查流程：
  1. 构建完整的14787 × 6505 稀疏矩阵 （可能耗时）
  2. 对每一列找最大值 （一行代码，最快）
  3. 检查约束条件
优势：   最快（如果矩阵构建不是瓶颈）
劣势：   内存可能不够，矩阵构建本身耗时


【SciPy vs NumPy vs Python 速度对比】
  Python循环：         ×1
  NumPy向量化：        ×10～100
  SciPy C函数：        ×100～1000
  GPU（如CuPy）：      ×1000～10000

这就是为什么SciPy这么快！
"""
